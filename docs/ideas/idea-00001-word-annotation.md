# Idea Brief

## One-line Summary

一个支持自定义词库高亮、排除规则和持久化标注的文档阅读与学习工具，帮助用户在 PDF、Markdown 等文本文件中高效识别重点词汇并沉淀个人笔记。

## Problem

用户在阅读英语学习材料、技术文档或考试资料时，往往需要手动识别目标词汇、区分已掌握与未掌握内容，并记录自己的理解与备注。现有阅读工具通常无法同时满足“按自定义词库精准高亮”“排除不需要的词”“对任意内容进行持久化标注”这几个需求，导致阅读、学习和复习过程割裂、低效。

## Target User

最明显会遇到这个问题的用户包括：

* 英语学习者，例如备考 CET4、CET6、考研、托福、雅思的人
* 需要精读大量英文资料的学生和研究者
* 阅读英文技术文档、论文、规范文档的开发者和专业人士
* 希望建立个人词汇和批注沉淀体系的重度阅读用户

## Current Alternatives

他们现在通常会这样解决：

* 使用 PDF 阅读器自带的手动高亮和批注功能
* 使用墨墨背单词、欧路词典、Anki 等词汇工具，但这些工具通常不直接结合原文阅读场景
* 使用 Notion、Obsidian、MarginNote 等工具做摘录和笔记
* 手工对照单词书或词表阅读，靠人工判断哪些词需要重点关注

## Proposed Solution

我的方案提供一个面向“词汇驱动阅读”的文档阅读产品，核心能力包括：

* 用户可预先配置多个词库，例如 CET4、CET6、考研、专业术语库，也可配置排除词库
* 支持上传 PDF、Markdown 等文本类文件，并对内容进行解析
* 系统根据用户启用的词库自动高亮命中的单词；若某词同时属于排除词库，则不高亮
* 用户可在文档中选中任意内容进行标注和注释
* 所有高亮状态、词库配置、标注内容都可以持久化，方便下次继续阅读和复习
* 后续还可以扩展为按词库切换视图、统计生词密度、导出标注结果等

## Why Better

相比现有替代方案，这个方案的优势是：

* 从“文档阅读”与“词汇管理”一体化出发，不需要在多个工具之间来回切换
* 支持自定义词库和排除规则，比普通阅读器的手动高亮更自动化、可控、可复用
* 标注内容可持久化，适合长期学习和反复回顾
* 不局限于英语考试词表，也可以扩展到专业术语、公司知识库、行业黑白名单词汇等场景
* 能把“阅读行为”转化为结构化学习资产，而不只是一次性的标记

## Why Now

为什么现在值得做：

* 用户越来越习惯把阅读、学习和知识管理整合到一个数字化工作流中
* 大模型和文本解析能力的成熟，使文档结构识别、词汇匹配、标注存储和后续智能扩展更容易实现
* 英语学习、专业阅读、知识管理这几个需求本来就高频存在，但目前缺少一个真正围绕“词库高亮 + 阅读标注”深度结合的产品
* 这个产品可以先从一个很聚焦的 MVP 做起，验证成本相对可控，后续扩展空间很大

## Risks

* 风险 1：PDF 解析质量不稳定，不同格式文档可能影响高亮准确性和选区体验
* 风险 2：单纯“高亮词汇”价值可能不足，若没有配套复习、统计或学习闭环，用户留存可能有限
* 风险 3：与通用阅读器、笔记软件、背词软件存在一定功能交叉，产品定位需要足够清晰
